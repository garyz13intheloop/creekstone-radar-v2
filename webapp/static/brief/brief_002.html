LangSmith Engine（公开 Beta）
LangChain · @hwchase17 · 2026-05-13 Interrupt 大会发布
Agent 可观测性
本期 hwchase17（♥251）宣布 LangSmith Engine 将在 Interrupt 大会发布，「用 Agent 调试 Agent」——监控生产 traces，聚类失败，识别根因，自动起草 PR 修复。
核心功能

LangSmith Engine 把 Agent 开发的核心循环（debug → eval → fix → prevent regression）完全自动化：

① Trace 监控：持续观察生产环境中的 Agent 执行日志
② 失败聚类：把相似失败自动归为一个「named issue」（如「Agent fails to handle subscription cancellation requests accurately」，影响 12% 会话）
③ 根因诊断：读取代码库，定位问题的具体代码位置（如发现工具描述歧义）
④ 自动修复 PR：起草具体的代码修改建议，人工 review 后合并
⑤ Eval 建议：为这类失败添加持续监控的测试用例

与 Anthropic Dreaming 的关系

Anthropic Dreaming（05-06 发布）是「事后回顾、写入记忆」；LangSmith Engine 是「实时监控、主动修复」。两者互补——前者让 Agent 通过经验积累变得更好，后者让开发者更快发现和修复 Agent 的问题。合在一起，是「Agent 持续改进」的两个核心支柱。

机构文章 · a16z
Your Data Agents Need Context — a16z · Jason Cui & Jennifer Li
a16z 基础设施团队定义了「AI 时代数据栈的新核心」：Context 层。过去十年建好的现代数据栈（Snowflake/dbt）解决了「数据放哪里、怎么查」，但 Agent 无法有效使用这些数据——因为它不知道「revenue 字段指的是 GAAP 还是 bookings」。Context 层（语义定义 + 数据血缘 + 访问策略 + 知识图谱）是所有数据 Agent 的共同前提依赖。
▸三代数据栈框架：孤岛时代 → 现代数据栈（第二代） → Agent-ready 语义层（第三代，正在发生）
▸a16z 原文：「每个正在构建数据 Agent 的公司，都会在 6 个月内遭遇同一堵墙：Context 缺失。Context 层是整个类别的基础设施，不是某个产品的差异化功能。」
▸投资方向：a16z 在寻找「从零为 Agent 设计的 Context Layer 产品」，而非传统数据目录改造的版本
问题的本质
「Over the past year, the market has realized that data and analytics agents are essentially useless without the right context – they aren't able to tease apart vague questions, decipher business definitions, and reason across disparate data effectively.」

翻译：「过去一年，市场已经意识到，数据和分析 Agent 在没有正确上下文的情况下基本上是没用的——它们无法拆解模糊的问题、解读业务定义，也无法有效地跨不同数据进行推理。」

Context 层的四个组成

Semantic Layer：业务指标的精确定义。「revenue = GAAP ASC 606 标准下的已确认收入，按季度统计，排除内部交易...」没有这一层，Agent 查到的数字对应什么含义是不确定的。

Lineage Graph：数据血缘。这个数字从哪来？经过哪些变换？被哪些报表依赖？Agent 发现异常值时，需要知道是哪条 pipeline 生成的。

Policy Layer：权限策略。谁能看哪些数据？在什么条件下？传统 RBAC 不够用，需要「Agent 意图感知」的权限系统。

Knowledge Graph：实体关系。客户 → 合同 → 产品 → 订单 → 发票 → 收入。Agent 回答「客户 A 健康状况」需要理解这条关系链。

a16z 的投资判断
「Every company building a data agent will hit the same wall within six months: context. The companies that solve this will be the picks and shovels of the agentic data economy. This is not a feature for a single product. It is infrastructure for an entire category.」
