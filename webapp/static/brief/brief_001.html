本期最强
Google Cloud Next '26 全面押注「Agentic Enterprise」：260 项发布，$750M Agent 生态基金，Gemini Enterprise Agent Platform 重构企业 AI 栈
Google Cloud Next '26（拉斯维加斯，Mandalay Bay，32,000+ 参会者）是本期最大信号。核心发布：① Gemini Enterprise Agent Platform（统一 Agent 构建、扩展、治理、优化）；② Agentic Data Cloud（跨云 Lakehouse + Knowledge Catalog）；③ Agentic Defense（Google Security Operations + Wiz 整合）；④ $750M 基金扶持 Agent 生态创业公司。

Bain & Company 分析：「Google assembled an agentic enterprise control plane」——Google 在用一套完整的 Agent 操控平台试图超越 OpenAI 和 Anthropic 在企业端的 harness 布局。
Google 终于有了「compelling harness」（Stratechery 此前说 Google 有好模型但没好 harness 是最大弱点）。这直接回应了 Stratechery 「Agents Over Bubbles」文章中的判断。Anthropic + OpenAI 的 harness 先发优势正在被 Google 的企业销售网络和 Cloud 生态快速压缩。

🚀 重大产品发布
Google Cloud Next '26 · Gemini Enterprise Agent Platform
Google · 2026-05-07 · 32,000+ 参会者 · 260 项发布
Agent 平台
统一的 Agent 构建 + 扩展 + 治理 + 优化平台。新功能：Agent Designer（图形化 Agent 构建）、Inbox（Agent 活动管理中心）、Long-running agents、Skills（可复用 Agent 能力模块）、Projects。同期发布 $750M 生态基金支持在 Google Cloud 上构建 Agent 的创业公司，可通过 Gemini Enterprise 和 Google Cloud Marketplace 直接分发产品给企业客户。
三大架构支柱

Gemini Enterprise Agent Platform：从零开始构建的 Agent 管理平台，不是把现有工具打包——有原生的 Agent 调试、版本管理、A/B 测试能力。与 Anthropic Managed Agents 的核心差异：Anthropic 提供的是 Agent runtime（Dreaming + Outcomes），Google 提供的是完整的 Agent 操控界面（Dashboard + SDK + Marketplace）。

Agentic Data Cloud：解决 a16z 文章中提到的「数据 Agent 需要 Context」问题。Cross-cloud Lakehouse 让 Agent 可以跨 AWS/Azure/Google Cloud 访问数据；Knowledge Catalog 提供语义层（类似 a16z 定义的 Context Layer）。

Agentic Defense：结合 Google Security Operations + Wiz（Google 2025 年以 $32B 收购）。AI 驱动的威胁狩猎 Agent + 检测工程 Agent + 第三方上下文整合。这是 Microsoft MDASH 的 Google 版本，但更偏向 SecOps 工作流而非漏洞发现。

Google positioned Agentic Defense as a central part of the enterprise AI stack.」

🔬 技术前沿
重磅模型发布
Poolside Laguna：从零训练 30 万亿 token，为 Agentic Coding 专门设计的 MoE 架构
Laguna M.1（225B/23B active MoE）= 目前 agentic coding 方向参数效率最高的专用模型之一。SWE-Bench Verified 72.5%（与 Claude Opus 4.7 相近）。关键设计选择：① MoE 架构——低激活参数（23B）意味着推理成本接近 23B 模型而能力接近 225B 模型；② 从头训练而非 fine-tune——30T token 都是 coding 相关内容，vs 通用 LLM 用大量非 coding 数据训练后再 fine-tune；③ 长周期任务专项——训练数据包含大量多步骤任务而非单步 Q&A。
Poolside 的路线印证了 Sarah Guo 的预测：coding 是第一个专门训练 Agent 模型的垂直。Laguna 的成功（如果能在用户场景中验证）将加速法律/医疗/金融方向的「专用 Agent 模型」出现。
大会 · 学术信号
ICLR 2026（里约热内卢，4月22日~26日）：AgentFlow + Flow-GRPO 是当前最重要的 Agent 训练突破
Lambda AI 的关键发现：7B 参数 Agent 通过 Flow-GRPO 超越 GPT-4o。Flow-GRPO 把长轨迹分解成单步更新，解决了「哪一步导致最终结果好/坏」的信用分配难题。另一篇重磅：UCF「Seeing, Listening, Remembering, and Reasoning」——多模态 Agent 的跨会话长时记忆架构。

→ Lambda AI ICLR 2026 报告
如果 AgentFlow 可产业化，7B 模型超越 GPT-4o 意味着垂直 Agent 的训练成本将大幅降低——不需要调用 frontier 模型，可以自己训练专用小模型，护城河更深。

Google Cloud Next '26 Keynote：The Agentic Enterprise Control Plane
Sundar Pichai + Thomas Kurian（Google Cloud CEO）的主题演讲核心内容。Google 的官方框架：我们正在进入「Agentic Era」，企业 AI 从「实验阶段」转向「执行阶段」，从「standalone copilots」转向「agent-led operating models」。这不只是产品发布，是 Google 对企业 AI 下一个 5 年的叙事定义。
▸Gemini Enterprise 定位：「an AI-native architecture connecting data, people, applications, and agents」——不是 AI 功能的集合，而是一套为 Agent 时代重新设计的企业操作系统

Bain & Company 的战略解读
「Google positioned Agentic Defense as a central part of the enterprise AI stack, with new Google Security Operations agents for threat hunting, detection engineering, and third-party context, as well as tighter integration with Wiz. The broader security story included Wiz AI Application Protection Platform, Wiz Security Agents, Wiz Workflows, Model Armor integration with Agent Gateway.」

翻译：Google 把安全放在 Agent 企业栈的核心位置——不是附加功能，而是架构级别的前提条件。这说明 Google 已经认识到「企业不会大规模部署 Agent，除非安全问题先被解决」。这与 OpenAI Daybreak 和 Microsoft MDASH 的时机选择形成了一个行业共识的信号：AI 安全是 2026 年企业 AI 采纳的关键瓶颈。

260 项发布里最重要的 3 项

① Gemini Enterprise Agent Platform 的 Agent Designer：图形化构建和测试 Agent，不需要写代码，类似 Zapier 但面向企业级 Agent。这会极大降低非技术用户构建 Agent 的门槛。

② Long-running agents：Google 的 Managed Agents 现在支持持续数天/周的任务，而不是单次会话——直接对标 Anthropic 的 Managed Agents。

③ Agentic Defense + Wiz 整合：把 $32B 收购的 Wiz（云安全）和 Google Security Operations（SOC 自动化）与 Agent 能力深度整合。这是 Google 到目前为止在安全领域最有实质内容的一次发布，可能改变企业安全软件市场格局。
