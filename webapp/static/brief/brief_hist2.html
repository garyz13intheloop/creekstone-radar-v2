OpenAI Harness Engineering 文章
Ryan Lopopolo（OpenAI Frontier）· Latent Space 播客同期发布
工程方法论
OpenAI 内部 Frontier 团队 Ryan Lopopolo 发表《Harness Engineering》长文（发布于 openai.com），同期做客 Latent Space 播客（72 分钟）。内容：如何在 OpenAI 内部用 Codex 构建并维护一个 100 万行代码的系统，0 人工写代码，0 人工 code review，每天消耗 1B+ tokens（$2-3k/天）。新名词：「Token Billionaires」。这是「Harness Engineering」概念的官方首次系统阐述。
Harness Engineering 的核心原则

Ryan 的核心论点：「当 Agent 失败时，不要 re-prompt 或让它重试——而是问：缺少了什么能力、context 或结构？」这个转变让团队持续改进 harness，而不是修 Agent 的行为。

关键产物：Symphony（github.com/openai/symphony）——一个「幽灵库」，是 Elixir 写的参考实现，为大规模 Codex Agent 系统提供脚手架，每个 Agent 都有 PRD 级别的完整 spec，但没有具体实现（由 Codex 生成）。

Ryan 的「borderline negligent」宣言
「If you're not burning through more than 1 billion tokens per day, you might be being borderline negligent about what you could be accomplishing. At market rates with caching assumptions, that's roughly $2-3k/day — a trivial expense compared to the engineering output.」

这是一个极其强烈的立场——他在说不激进使用 AI 是失职。对应的数字：1B tokens/天 ≈ $2-3k/天 ≈ $60-90k/月。对于一个工程团队，这相当于 1-2 个工程师的工资，但产出可能是 10 倍人工的代码量。

📚 深度阅读
🎙 播客 · Latent Space
Extreme Harness Engineering for Token Billionaires — Ryan Lopopolo, OpenAI Frontier & Symphony
swyx 和 Alessio 采访 Ryan Lopopolo（OpenAI Frontier 团队），首次公开揭示 OpenAI 内部「Dark Factory」的完整运作方式：100 万行代码，0 人工代码，0 人工 code review，每天 1B+ tokens。这期播客与 Ryan 同期发布的 openai.com 长文《Harness Engineering》配合，成为 2026 年 Q2 最重要的 AI 工程思想材料之一。
▸Ryan 的核心转变：当 Agent 失败时，不 re-prompt，而是问「缺少了什么能力、context 或结构？」——这逼迫团队持续改进 harness 而非修 Agent
▸Symphony（开源）：为大规模 Codex Agent 系统提供脚手架的「幽灵库」，每个 Agent 有 PRD 级别的 spec 但无实现（Codex 生成）
▸Ryan 的「borderline negligent」宣言：「不用到 1B tokens/天，可能是在失职」——这是 AI 激进主义在工程层面的最强表达之一
▸软件的新写法：「写给模型看的，而不只是写给工程师看的」——代码越来越需要对 Agent 友好（清晰的接口、明确的副作用）
Ryan Lopopolo 原话（来自 Latent Space 播客）
「I would say that if we think that we want these systems to do more and more of the work, we should be thinking about what is the right agent to run for a particular task given where it sits on those dimensions. The real bottleneck in AI-native software development is now human attention rather than tokens.」

翻译：「我认为，如果我们想让这些系统做越来越多的工作，我们应该思考的是：对于特定任务，根据它在这些维度上的位置，什么是正确的 Agent？AI 原生软件开发的真正瓶颈现在是人类的注意力，而不是 token。」

这句话非常重要：从「token 是瓶颈」（2024 年的判断）到「人类注意力是瓶颈」（2026 年的判断），代表了整个 AI 使用范式的根本转变。

swyx 的解读
「Move over, context engineering. Now it's time for Harness Engineering and the age of the token billionaires.」

这句话把 Ryan 的文章和播客定位为「比 Prompt Engineering → Context Engineering 更重要的下一个阶段」。swyx 拥有极高的社区话语权，他的这个命名将推动「Harness Engineering」作为一个独立的工程职能被更广泛认可。
