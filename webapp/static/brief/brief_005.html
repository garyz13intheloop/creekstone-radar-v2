企业信号
Chollet 的「生产力悖论」vs Anthropic「75% AI 代码 + 继续招人」：AI 提升了代码量但未必提升了价值
本期出现了两个看似矛盾的企业信号：
① Anthropic 内部：75% 代码由 AI 生成，同时仍在大规模招聘工程师（多家媒体报道）
② @fchollet（♥1,778）：「The quantity of code that devs ship has roughly 10xed. But net developer productivity (value created by unit of time) is only up by a bit, if at all.」

两者合在一起描绘了一个真实图景：AI 让代码量爆炸（10x），但协调成本、维护成本、调试 AI 生成错误的时间也相应增加，吸收了大部分生产力提升。Anthropic 继续招聘是因为工程师的角色变了（架构设计 + Agent 编排 + 质量监控），而不是消失了。

@fchollet 第二条（♥1,262）：「A mental model for working with coding agents is that they're blind squirrels running into a maze and bumping into walls.」——agent 需要「constrained context」（约束上下文）而不是「more freedom」（更多自由）。
这两个信号对早期 AI 创业公司有直接含义：① 「AI 直接替代工程师」的叙事被实际数据否定——工程师数量在增加；② 真正的机会在「提供更好的约束和上下文」而非「更强的模型自由度」——Agent Harness 工具比「更强的 base model」更容易产生实际价值。

François Chollet
@fchollet · Keras 作者 · ARC Prize 创始人
♥ 1,340
05-17
「Decision making was the bottleneck all along. Productivity is the rate at which you make open-ended decisions, the rate at which you reduce future paths.」
这是 Chollet 的「生产力重新定义」。在 AI 时代，代码生成已经被自动化（10x 代码量），但决策仍然是人类的。Chollet 的论点：真正的生产力瓶颈从来不是「写代码的速度」，而是「做开放性决策的速度」——什么功能重要、怎么架构系统、如何权衡 tradeoff。AI 提高了代码速度，但没有提高决策速度，所以净生产力（真实价值创造）增长有限。

结合他的第二条（♥1,262）「blind squirrels running into a maze」：Coding Agent 就像盲松鼠在迷宫里乱撞，它需要的不是更大的迷宫（更多自由），而是更好的约束（verifiable constraints）。这两条帖子合在一起，给出了「如何设计 Agent 系统」的根本逻辑：① 减少 Agent 需要做的开放性决策，让它只在已定义好的约束内工作；② 提供清晰的验证条件，让 Agent 知道什么叫「完成了」。

认知框架
Chollet 双帖：「决策是生产力瓶颈」+「Agent = 需要约束的盲松鼠」——给 Agent 设计最重要的两个原则
@fchollet 本期两条关键帖合在一起，构成了目前对「如何正确使用 AI Coding Agent」最精炼的框架：
① ♥1,340：「Decision making was the bottleneck all along. Productivity is the rate at which you make open-ended decisions.」—— AI 提高了代码速度，但真正的生产力瓶颈是「开放性决策速度」，这个 AI 没有改变。
② ♥1,262：「A mental model for working with coding agents is that they're blind squirrels running into a maze and bumping into walls.」——Agent 最大的挑战不是模型能力，而是「给 agent 合适的约束上下文（constrained context）」。信息太多或来源冲突，Agent 就会产生错误输出。

「决策瓶颈」框架的实用含义

如果决策是生产力瓶颈，那么「有助于做更好的决策」比「有助于执行决策」更有价值。这对 AI 工具的产品设计有直接影响：帮助用户做决策的 AI 工具（如「这个架构决策有哪些 tradeoff？」「这 5 个备选方案哪个最符合我们的约束？」）比「帮用户写更多代码」的 AI 工具更有长期价值。

「盲松鼠」框架的工程含义

Chollet 的原文被 Digg 引用的关键段落：「Probably the biggest challenge that most companies run into in their agent strategy is getting agents the right constrained context to work with for a task. Too much information or conflicting sources, and the agent can easily draw from the data and produce the wrong result.」

这句话是对「给 Agent 更大的 context window」这条路线的反驳——更大的 context 不等于更好的结果，关键是 context 的质量和约束性。clawpatch 把代码库分割成语义单元、Hermes /goal 的 judge 模型、开源 Claude Code harness 的结构化约束——这些都是对「盲松鼠需要约束」原则的工程实现。

对于 Agent 系统设计：不要给 Agent 「更多信息」，要给 Agent「更好的约束和更清晰的验证条件」。这是 2026 年 Agent 工程化成熟的核心认知转变。

Applied Intuition「Physical AI」：$15B 公司，从 YC 工具公司到自动驾驶 OS，「车辆是 Android 出现前的手机」
Latent Space（04-27 发布，本期热播）采访 Applied Intuition CEO Qasar Younis + Peter Ludwig。核心框架：「Physical AI is not just LLMs on wheels」——真正的瓶颈不再是模型智能，而是「在受限硬件上的可靠部署」。Applied Intuition 的三大技术支柱：① 仿真和 RL 基础设施；② 安全关键机器的真实操作系统；③ 自主性和世界理解的基础 AI 模型。L4 自动驾驶卡车已在日本实际运营。Coding Agent 内部应用：使用 Cursor、Claude Code 等，有内部采用排行榜。

为什么「物理 AI」不同于「屏幕 AI」

Latent Space show notes 原文：「Learned systems can make mistakes in chat or coding, but safety-critical machines like driverless trucks, autonomous vehicles, and robots need much higher reliability.」

物理 AI 有一个根本的不对称性：LLM 在 chat 里犯错的代价是「给了错误答案，用户纠正」；物理机器犯错的代价可能是「人员伤亡」。这个可靠性要求使物理 AI 的技术门槛高出一个数量级。

「车辆是 Android 出现前的手机」

Peter Ludwig 的类比：「Today's vehicle and machine software stack is fragmented across many operating systems」——就像 2007 年前每部手机都有自己的操作系统（Symbian、Windows Mobile、Palm OS 等）一样，今天的车辆/机器也有数十种不同的 OS（ROS、AUTOSAR、各种专有系统）。Applied Intuition 想做「车辆界的 Android/iOS」——提供统一的物理 AI 操作层。

为什么 tooling 公司回来了

Qasar：「Developer tooling looked unfashionable in 2016, but Applied Intuition still bet on it. The AI boom made workflows and tools central again.」——2016 年 VC 不看好 tooling 公司（投资更看好直接面向消费者的产品），Applied Intuition 逆势而上，现在被证明是正确的。这对 Creekstone 的「Agent 基础设施投资」有直接的历史参照意义。

物理 AI（自动驾驶、机器人、工业机器）是 2026 年最被低估的 AI 赛道之一。Applied Intuition $15B 估值证明了这个赛道的商业价值，但大多数 AI 投资者仍然关注软件 Agent，对物理 AI 的关注度不足。
