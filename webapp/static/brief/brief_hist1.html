Sequoia AI Ascent 2026：Karpathy「Software 3.0 + Agentic Engineering」重新定义程序员的角色，Demis Hassabis 谈 AGI 时间线
Sequoia AI Ascent 2026（Sequoia 年度 AI 峰会，邀请 150+ 顶级创始人）于本期举行。阵容：Greg Brockman（OpenAI），Andrej Karpathy，Demis Hassabis（Google DeepMind CEO），Boris Cherny（Anthropic Claude Code 负责人），Dmitri Dolgov（Waymo CEO）。

Karpathy 在峰会上发表了他 2026 年最重要的演讲，并在 04-30 发布了完整博客版本。核心论点：「2025 年 12 月是 Agentic 的拐点。Software 3.0 = LLM 是新的可编程层。程序员正在从代码撰写者变成 Agent 编排者。」
Sequoia AI Ascent 是 Sequoia 向顶级 founder 传递投资判断的年度渠道。Karpathy 的 Software 3.0 框架将被整个 VC 圈采纳——Creekstone 的内容和投资逻辑应该主动对齐这个叙事框架。

模型发布
Claude Opus 4.7 发布：SWE-Bench 87.6%，新增 xhigh 推理等级，视觉能力大幅提升，Claude Design 同期发布
Anthropic 04-16 发布 Claude Opus 4.7（本期开始大规模用户使用），这是当时公开发布的最强 AI 模型之一。关键数据：SWE-Bench Verified 87.6%（vs 4.6 的 58%，大幅跳升），SWE-Bench Pro 64.3%，CursorBench 70%，BixBench 0.751 pass@1（超越 GPT-5.4、Grok 4.2、Gemini 3.1 Pro）。新功能：xhigh 推理等级（介于 extended thinking 和 normal 之间的精细控制），更强的视觉和多模态能力。价格保持 $5/$25 per million token（与 4.6 相同）。同期发布 Claude Design（把对话转化为原型、PPT、营销材料）。
Claude Opus 4.7 的 SWE-Bench 从 58% 跳到 87.6% 是一个质变。这不是渐进改进，是能力层级的切换。在 Coding Agent 场景，87.6% 意味着大多数实际软件工程任务可以被可靠完成，而不只是演示。

Claude Opus 4.7 用户画像：80% 的高频用户来自 $10 万+ 家庭收入——AI 效益呈现「知识工作者精英化」趋势
Ethan B. Holland 引用数据：80% 的 Claude 高频用户来自年收入 $10万+ 的家庭，vs Meta AI 用户的 37%。这说明 Claude 当前的用户结构是「高收入知识工作者」为主，而非大众消费者。对比 ChatGPT 的更广泛用户基础，Anthropic 的用户质量更高（更高 LTV）但渗透率更低。

xhigh 推理等级

新增的 xhigh effort level 让开发者可以在「更好的推理质量」和「更低的延迟」之间做更精细的权衡，而不只是 on/off 的 extended thinking 开关。对于 Agent 系统，这意味着可以根据任务的复杂度动态调整推理深度，而不是每次都跑最贵的模式。

Claude Design

同期发布的 Claude Design 是 Anthropic Labs 的一个实验性产品：把对话转化为原型、幻灯片和营销材料，读取用户代码库的设计系统后交给 Claude Code 实现。这是 Anthropic 第一次进入「设计工具」领域，被一些分析者视为「对 Figma 和 Lovable 的直接攻击」。

📚 深度阅读
Karpathy 博客 · 必读
Sequoia Ascent 2026 Summary — Andrej Karpathy（Software 3.0 + Agentic Engineering 完整版）
Karpathy 的 Sequoia AI Ascent 演讲完整博客版，包含 AI 生成的演讲摘要和清理后的逐字稿（用 Codex 5.5 生成）。这是目前对「软件开发如何被 Agent 重塑」最系统、最有说服力的第一手阐述。对于 Creekstone 理解垂直 Agent 的投资逻辑，这篇是必读材料。
▸拐点时刻：「2025 年 12 月是 Agentic 拐点。我从来没有感到如此落后过——不是因为编程更难了，而是默认工作流改变了。」生成的代码块更大、更连贯、更可靠，Karpathy 开始信任 Agent 处理更大的任务单元。
▸Software 3.0：Software 1.0（人写代码）→ 2.0（人创建数据集和目标，神经网络学习）→ 3.0（人通过 prompt、context、工具、示例、记忆、指令来编程 LLM）。「context window 成为主要杠杆」。
▸软件正在消失：MenuGen 案例——一个需要前端+API+图片生成+部署+认证的 app，在 Software 3.0 下变成「一个照片给多模态模型，直接输出菜品图片叠加在菜单上」。「AI 不只是更快构建旧 app 的方法，有些 app 应该停止存在。」
▸可验证性（核心框架）：「传统软件自动化你能指定的，LLM 自动化你能验证的。」编程 Agent 效果好因为测试通过/失败是清晰的验证信号。这解释了为什么 coding 是第一个被完全 Agent 化的专业。
▸锯齿状智能：capability spike ≈ 验证性 × 训练注意力 × 数据覆盖 × 经济价值。AI 在某些任务上超人类是因为这个任务「被重点训练」，而非「智能整体提升」。这个框架对评估 AI 模型的真实能力非常重要。
第 3 节：MenuGen 和软件消失的时刻（完整）
「I used MenuGen as an example of a deeper shift. MenuGen was a traditional web app: take a picture of a restaurant menu, OCR the dish names, generate images of the dishes, and render the result in a UI. It required frontend code, APIs, image generation, deployment, auth, payments, secrets, and infrastructure. But later, I saw the Software 3.0 version: take a photo of the menu, give it to a multimodal model, and ask it to render dish images directly onto the menu image. In that version, much of the app disappears. The neural network directly transforms input media into output media. The old software stack was scaffolding around a transformation the model can now perform directly. This is one of the most important founder implications: AI is not just a faster way to build the old apps. Some apps should stop existing as apps.」

翻译：「我用 MenuGen 来举例说明一个更深层的转变。MenuGen 是一个传统 web app：拍下餐厅菜单的照片，OCR 菜品名，生成菜品图片，在 UI 中渲染。它需要前端代码、API、图片生成、部署、认证、支付、秘钥和基础设施。但后来，我看到了 Software 3.0 版本：拍下菜单照片，交给多模态模型，让它直接把菜品图片叠加在菜单图片上渲染出来。在那个版本里，大部分 app 消失了。神经网络直接把输入媒体变成输出媒体。旧的软件栈是围绕模型现在可以直接完成的变换搭建的脚手架。这是对创始人最重要的含义之一：AI 不只是更快构建旧 app 的方法。有些 app 应该停止存在为 app。」

第 5 节：可验证性框架（完整）
「My core automation framework is: Traditional software automates what you can specify. LLMs and reinforcement learning automate what you can verify. If a task has an automatic reward or success signal, models can practice it. This is why math, coding, tests, benchmarks, games, and many engineering tasks improve so quickly. They are resettable, repeatable, and rewardable. This also explains why coding agents feel dramatically better than many ordinary chatbot experiences. Coding gives the model feedback: tests pass or fail, programs run or crash, diffs can be inspected, benchmarks can be measured.」

翻译：「我的核心自动化框架是：传统软件自动化你能指定的。LLM 和强化学习自动化你能验证的。如果一个任务有自动的奖励或成功信号，模型就可以练习它。这就是为什么数学、编程、测试、基准测试、游戏和许多工程任务改进如此迅速。它们是可重置的、可重复的、可奖励的。这也解释了为什么 coding agent 感觉比许多普通的聊天机器人体验好得多。编程给了模型反馈：测试通过或失败，程序运行或崩溃，diff 可以被检查，基准可以被测量。」

对垂直 Agent 的含义：「可验证性」框架是选择垂直 Agent 赛道的核心判断工具。一个任务越容易验证（有清晰的成功/失败标准），AI 越快变得有用。法律合同审查（有明确的条款检查列表）> 战略咨询（结果很难在短期验证）。

机构文章 · Sequoia
Sequoia Capital：「2026: This Is AGI」
Sequoia 的 2026 年 AI 报告（年度核心观点文章）。核心论点：「Long-horizon agents are functionally AGI, and 2026 will be their year.」Sequoia 不是在讨论 AGI 的哲学定义，而是在说长周期 Agent 在功能意义上已经达到了 AGI——它可以「figure things out」，用有限知识+推理能力+迭代能力完成复杂任务。
▸Sequoia 的 AGI 定义：「AGI is the ability to figure things out. A human who can figure things out has some baseline knowledge, the ability to reason over that knowledge, and the ability to iterate their understanding.」——这是一个工程化的定义，不是哲学定义
▸MAD Landscape 2026（Matt Turck）预计在 Q2 发布：将是今年最重要的 AI 行业地图，追踪所有 ML/AI/Data 公司的状态
